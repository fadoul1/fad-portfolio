## Commands

```bash
npm start          # Start local dev server (http://localhost:3000)
npm run build      # Production build → ./build/
npm run serve      # Serve the production build locally
npm run clear      # Clear Docusaurus cache and build artifacts
npm run typecheck  # Run TypeScript type checking (tsc)
npm run deploy     # Deploy to GitHub Pages
```

No test framework is configured.

## Architecture

This is a **Docusaurus 3.x** static site (portfolio/documentation) using TypeScript and React 19.

**Key config files:**
- [docusaurus.config.ts](docusaurus.config.ts) — site metadata, navbar, footer, plugins, theme
- [sidebars.ts](sidebars.ts) — docs sidebar (autogenerated from `docs/` folder structure)
- [src/css/custom.css](src/css/custom.css) — Infima CSS variable overrides for light/dark theming

**Content directories:**
- `docs/` — Documentation pages; folder structure drives the sidebar via `_category_.json` files
- `blog/` — Blog posts; authors defined in `blog/authors.yml`, tags in `blog/tags.yml`
- `static/` — Static assets served at root (images, favicons, etc.)

**React/TypeScript source:**
- `src/pages/` — Full custom pages (each file becomes a route)
- `src/components/` — Shared React components
- Component styles use CSS Modules (`*.module.css`)

**Content authoring:** Pages can be `.md`, `.mdx` (MDX = Markdown + JSX), or `.tsx`. MDX files can import and render React components inline.

**Theming:** Docusaurus uses the Infima CSS framework. Override CSS variables in `custom.css` under `:root` (light) and `[data-theme='dark']` (dark). To override built-in Docusaurus components, use `npm run swizzle`.
